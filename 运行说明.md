# 运行说明

## 开发环境运行

```bash
npm install
npm run dev
```

访问：http://localhost:3000

## 生产环境构建

```bash
npm run build
npm start
```

## 类型检查

```bash
npx tsc --noEmit
```

## 如何添加新的数据集

### 方法 1：直接添加 JSON 文件

1. 在 `json_data` 文件夹中创建新的 JSON 文件，例如 `history.json`
2. 按照以下格式编写内容：

```json
[
  {
    "id": 1,
    "question": "问题内容",
    "answer": "答案内容",
    "author": "作者名（可选）",
    "category": "分类（可选）"
  },
  {
    "id": 2,
    "question": "另一个问题",
    "answer": "另一个答案",
    "author": null,
    "category": "分类"
  }
]
```

3. 刷新页面，新数据集会自动出现在左侧边栏

### 方法 2：自定义显示名称

如果想自定义数据集的显示名称，可以修改 `app/api/datasets/route.ts` 中的 `nameMap`：

```typescript
const nameMap: Record<string, string> = {
  quotes: "智慧名言",
  poems: "古诗词",
  history: "历史知识", // 添加新的映射
  science: "科学常识", // 添加新的映射
};
```

## 注意事项

1. **ID 必须唯一**：每个 JSON 文件中的 `id` 字段必须在该文件内唯一
2. **文件名规范**：建议使用英文小写字母和下划线，例如 `my_dataset.json`
3. **JSON 格式**：确保 JSON 文件格式正确，可以使用在线工具验证
4. **自动加载**：添加或删除 JSON 文件后，刷新页面即可看到变化

## 功能说明

### 左侧边栏

- 显示所有可用的数据集
- 点击切换数据集
- 显示每个数据集的卡片数量
- 移动端支持折叠/展开

### 学习模式

- **专注模式**：一次显示一张卡片，支持键盘导航（← →）
- **列表模式**：网格显示所有卡片

### 学习进度

- 自动保存已学习的卡片
- 支持删除已掌握的卡片
- 进度条显示学习进度
- 数据存储在浏览器本地

### 其他特性

- 卡片顺序自动随机打乱
- 响应式设计，支持移动端
- 暗色主题界面
